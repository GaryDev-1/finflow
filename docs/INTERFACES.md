## Data Conventions

All interfaces in FinFlow share the following conventions:

### Monetary values

All monetary amounts are **integers representing cents** (South African Rand).

```
R150.00  →  15000
-R3,200  →  -320000
```

Formatting is exclusively the frontend's responsibility. Use the [`formatMoney`](#formatmoney) utility.

**References:**
- [Why store money as integers](https://stackoverflow.com/questions/3730019/why-not-use-double-or-float-to-represent-currency) — avoids floating-point rounding errors
- [Money scalar in GraphQL](https://www.apollographql.com/blog/backend/using-the-graphql-scalar-type-for-money) — Apollo blog on Money scalar design

### Timestamps

All timestamps are **ISO 8601 strings in UTC** (`TIMESTAMPTZ` in PostgreSQL).

```
"2025-01-15T09:30:00.000Z"
```

**References:**
- [ISO 8601 — Wikipedia](https://en.wikipedia.org/wiki/ISO_8601)
- [PostgreSQL TIMESTAMPTZ](https://www.postgresql.org/docs/current/datatype-datetime.html)

### Identifiers

All IDs are **UUID v4** strings generated by PostgreSQL's `uuid-ossp` extension.

```
"a1b2c3d4-e5f6-7890-abcd-ef1234567890"
```

**References:**
- [uuid-ossp extension](https://www.postgresql.org/docs/current/uuid-ossp.html)
- [UUID v4 — RFC 4122](https://datatracker.ietf.org/doc/html/rfc4122)

### Response envelope

Every REST endpoint returns one of two shapes:

**Success:**
```json
{
  "data": <payload>,
  "meta": {
    "total": 42,
    "page": 1
  }
}
```

**Error:**
```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "Account not found"
  }
}
```

`meta` is omitted from single-resource responses (e.g. `GET /accounts/:id`).

**Source:** [`services/accounts-service/src/db/migrations/001_init.sql`](../services/accounts-service/src/db/migrations/001_init.sql)

CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

CREATE TABLE IF NOT EXISTS accounts (
  id             UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id        VARCHAR(255) NOT NULL,
  account_number VARCHAR(50)  NOT NULL UNIQUE,
  account_type   VARCHAR(20)  NOT NULL CHECK (account_type IN ('CHEQUE','SAVINGS','CREDIT')),
  balance        INTEGER      NOT NULL DEFAULT 0,  -- cents
  currency       VARCHAR(10)  NOT NULL DEFAULT 'ZAR',
  status         VARCHAR(20)  NOT NULL DEFAULT 'ACTIVE'

  created_at     TIMESTAMPTZ  NOT NULL DEFAULT NOW(),

## Transactions Service — REST API

**Base URL (local):** `http://localhost:3002`
**Port:** `3002`

### TypeScript Types

**Source:** [`services/transactions-service/src/types/index.ts`](../services/transactions-service/src/types/index.ts)

```typescript
type TransactionType     = 'CREDIT' | 'DEBIT';
type TransactionCategory = 'SALARY' | 'TRANSFER' | 'PAYMENT' | 'WITHDRAWAL' | 'DEPOSIT' | 'FEE';
type TransactionStatus   = 'PENDING' | 'COMPLETED' | 'FAILED' | 'REVERSED';

interface Transaction {
  id:              string;              // UUID v4
  accountId:       string;             // UUID v4 — references an account
  type:            TransactionType;
  amount:          number;             // cents (integer, always positive)
  currency:        string;             // "ZAR"
  description:     string;
  category:        TransactionCategory;
  reference:       string;             // unique per transaction
  status:          TransactionStatus;
  transactionDate: string;             // ISO 8601
  createdAt:       string;             // ISO 8601
}

interface TransactionSummary {
  totalCredits:     number;  // sum of all CREDIT amounts in cents
  totalDebits:      number;  // sum of all DEBIT amounts in cents
  netBalance:       number;  // totalCredits - totalDebits in cents
  transactionCount: number;  // count of matched transactions
}
```

### Database Schema

**Source:** [`services/transactions-service/src/db/migrations/001_init.sql`](../services/transactions-service/src/db/migrations/001_init.sql)

```sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

CREATE TABLE IF NOT EXISTS transactions (
  id               UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  account_id       UUID         NOT NULL,
  type             VARCHAR(10)  NOT NULL CHECK (type IN ('CREDIT','DEBIT')),
  amount           INTEGER      NOT NULL,              -- cents, always positive
  currency         VARCHAR(10)  NOT NULL DEFAULT 'ZAR',
  description      TEXT         NOT NULL,
  category         VARCHAR(20)  NOT NULL
                   CHECK (category IN ('SALARY','TRANSFER','PAYMENT','WITHDRAWAL','DEPOSIT','FEE')),
  reference        VARCHAR(100) NOT NULL UNIQUE,
  status           VARCHAR(20)  NOT NULL DEFAULT 'COMPLETED'
                   CHECK (status IN ('PENDING','COMPLETED','FAILED','REVERSED')),
  transaction_date TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  created_at       TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_transactions_account_id ON transactions(account_id);
CREATE INDEX IF NOT EXISTS idx_transactions_date ON transactions(transaction_date DESC);
```

```typescript
type LoanStatus      = 'PENDING' | 'ACTIVE' | 'SETTLED' | 'DEFAULTED' | 'REJECTED';
type RepaymentStatus = 'UPCOMING' | 'PAID' | 'OVERDUE' | 'PARTIAL';

interface Loan {
  id:                 string;        // UUID v4
  userId:             string;        // e.g. "user-001"
  principalAmount:    number;        // original loan amount in cents
  outstandingBalance: number;        // remaining balance in cents
  interestRate:       number;        // annual rate as a decimal percent, e.g. 9.5
  termMonths:         number;        // loan term in months, e.g. 240
  monthlyInstalment:  number;        // fixed monthly payment in cents
  status:             LoanStatus;
  disbursedAt:        string | null; // ISO 8601, null if not yet disbursed
  nextPaymentDate:    string | null; // ISO 8601, null if no next payment
  currency:           string;        // "ZAR"
  createdAt:          string;        // ISO 8601
  updatedAt:          string;        // ISO 8601
}

interface Repayment {
  id:      string;             // UUID v4
  loanId:  string;             // UUID v4 — references a loan
  dueDate: string;             // ISO 8601
  amount:  number;             // cents
  status:  RepaymentStatus;
  paidAt:  string | null;      // ISO 8601, null if unpaid
}
```

### Database Schema

**Source:** [`services/loans-service/src/db/migrations/001_init.sql`](../services/loans-service/src/db/migrations/001_init.sql)

```sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

CREATE TABLE IF NOT EXISTS loans (
  id                  UUID         PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id             VARCHAR(255) NOT NULL,
  principal_amount    INTEGER      NOT NULL,    -- cents
  outstanding_balance INTEGER      NOT NULL,    -- cents
  interest_rate       NUMERIC(5,2) NOT NULL,    -- e.g. 9.50
  term_months         INTEGER      NOT NULL,
  monthly_instalment  INTEGER      NOT NULL,    -- cents
  status              VARCHAR(20)  NOT NULL DEFAULT 'PENDING'
                      CHECK (status IN ('PENDING','ACTIVE','SETTLED','DEFAULTED','REJECTED')),
  disbursed_at        TIMESTAMPTZ  NULL,
  next_payment_date   TIMESTAMPTZ  NULL,
  currency            VARCHAR(10)  NOT NULL DEFAULT 'ZAR',
  created_at          TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  updated_at          TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS repayments (
  id       UUID        PRIMARY KEY DEFAULT uuid_generate_v4(),
  loan_id  UUID        NOT NULL REFERENCES loans(id) ON DELETE CASCADE,
  due_date TIMESTAMPTZ NOT NULL,
  amount   INTEGER     NOT NULL,   -- cents
  status   VARCHAR(20) NOT NULL DEFAULT 'UPCOMING'
           CHECK (status IN ('UPCOMING','PAID','OVERDUE','PARTIAL')),
  paid_at  TIMESTAMPTZ NULL
);

CREATE INDEX IF NOT EXISTS idx_loans_user_id      ON loans(user_id);
CREATE INDEX IF NOT EXISTS idx_repayments_loan_id ON repayments(loan_id);
```

### Authentication

The BFF uses **JWT (HS256)** authentication via the `jose` library.

**References:**
- [jose library](https://github.com/panva/jose) — JWT implementation
- [RFC 7519 — JSON Web Tokens](https://datatracker.ietf.org/doc/html/rfc7519)
- [HS256 — HMAC-SHA256](https://datatracker.ietf.org/doc/html/rfc7518#section-3.2)

**Source:** [`apps/bff/src/middleware/auth.ts`](../apps/bff/src/middleware/auth.ts)

#### Token format

```
Authorization: Bearer <jwt>
```

#### JWT payload shape

```typescript
interface JWTPayload {
  sub: string;     // user ID, e.g. "user-001"
  iat: number;     // issued-at (Unix timestamp)
  exp: number;     // expiry (Unix timestamp)
}
```

#### Auth functions

```typescript
// Extract token string from Authorization header
function extractToken(authHeader: string | undefined): string | null

// Sign a new token (uses JWT_SECRET + JWT_EXPIRY env vars)
async function signToken(payload: Record<string, unknown>): Promise<string>

// Verify a token; returns payload or null (does not throw on invalid tokens)
async function verifyToken(token: string): Promise<JWTPayload | null>
```

#### Demo credentials

```
username: demo
password: demo123
```

> **Note:** The microservices themselves have no authentication. Auth is enforced only at the BFF layer. The BFF attaches the decoded `user` payload to the Apollo context, which resolvers can use to scope queries.

### Context

**Source:** [`apps/bff/src/context/index.ts`](../apps/bff/src/context/index.ts)

The context builder runs on every GraphQL request:

```typescript
async function buildContext({ req, res }: {
  req: express.Request;
  res: express.Response;
}): Promise<AppContext>
```

**What it does:**
1. Calls `extractToken(req.headers.authorization)` to get the raw JWT string
2. Calls `verifyToken(token)` — on failure, `user` is `null` (no error thrown)
3. Instantiates `AccountsAPI`, `TransactionsAPI`, `LoansAPI`
4. Returns `{ dataSources, user }`

- [Apollo Server context](https://www.apollographql.com/docs/apollo-server/data/context/)
